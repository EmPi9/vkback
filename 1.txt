Выбор базы данных и обоснование
Для реализации интернет-магазина крупного ритейлера можно использовать реляционную базу данных PostgreSQL
для хранения основной информации о товарах, пользователях и заказах. Документоориентированную базу данных MongoDB
можно использовать для хранения динамических данных о товарах (например, дополнительных полей товаров, которые могут различаться).

Реляционная база данных PostgreSQL
Плюсы:

Поддержка сложных запросов и транзакций.
Надежность и согласованность данных.
Простота интеграции с внешними системами.
Хорошо подходит для данных с четкой структурой.
Минусы:

Ограниченная гибкость для данных со сложной и изменяемой структурой.
Документоориентированная база данных MongoDB

Плюсы:

Гибкость в структуре данных.
Легкость масштабирования.
Хорошо подходит для хранения документов с динамическими полями.
Минусы:

Ограниченная поддержка сложных транзакций.
Может быть сложнее интегрироваться с другими системами.
Схема данных
Реляционная база данных (PostgreSQL)
Users (Пользователи):

id (PK)
name
email
password
city_id (FK)
Cities (Города):

id (PK)
name
Warehouses (Склады):

id (PK)
city_id (FK)
address
ProductGroups (Группы товаров):

id (PK)
name
image
Categories (Категории):

id (PK)
name
group_id (FK)
image
Subcategories (Подкатегории):

id (PK)
name
category_id (FK)
image
Products (Товары):

id (PK)
name
subcategory_id (FK)
image
price
quantity
ProductPrices (Цены на товары в разных городах):

id (PK)
product_id (FK)
city_id (FK)
price
Orders (Заказы):

id (PK)
user_id (FK)
status
created_at
OrderItems (Позиции заказа):

id (PK)
order_id (FK)
product_id (FK)
quantity
price
Документоориентированная база данных (MongoDB)
Products (Товары):
_id
name
subcategory_id
image
additional_fields (дополнительные поля)

Хранение медиаконтента
Медиаконтент (изображения товаров, групп и категорий) предлагается хранить в объектном хранилище, 
Amazon S3 или его аналоге, предоставляемом российскими провайдерами Яндекс.Облако или Mail.ru Cloud Solutions.

Плюсы:

Высокая доступность и отказоустойчивость.
Масштабируемость.
Простота интеграции с веб-приложением через URL.
Минусы:

Затраты на использование облачного хранилища.
Необходимость управления правами доступа к файлам.
